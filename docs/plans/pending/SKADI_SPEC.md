## 1. Что известно

SKADI упоминается в roadmap как планируемый компонент системы AIDA, но
на момент написания (16.04.2026) детальная спецификация не определена.

Название происходит из скандинавской мифологии (Skaði — богиня охоты и лыж).

---

## 2. Кандидаты на назначение

На основе существующих пробелов архитектуры, SKADI может закрывать одно из:

### Вариант A: Scheduler / Harvest orchestrator

**Назначение:** Управление расписанием автоматического парсинга (UC1 — scheduled harvest).

DALI сейчас выполняет scheduled harvest через JobRunr. SKADI мог бы выделить
scheduling logic в отдельный сервис:
- Cron-based и event-based триггеры
- Управление приоритетами очереди
- Retry и dead-letter queue
- Multi-tenant scheduling

### Вариант B: Schema diff service

**Назначение:** Сравнение двух состояний YGG (до/после парсинга).

- `diff(session1_id, session2_id)` → добавленные / удалённые / изменённые объекты
- Notification при критических изменениях (колонка удалена, routine переименована)
- Интеграция с HEIMDALL для алертинга

### Вариант C: Source control integration

**Назначение:** Подключение к Git (GitHub/GitLab/Bitbucket).

- Webhook при push → автоматический trigger parse session
- Diff-парсинг только изменённых файлов
- Линковка `DaliStatement` → Git commit + file + line number
